# Frequently Asked Questions (FAQ)

Answers to common questions about Atrium's features, stack, and local execution based strictly on the current codebase implementation.

---

## 1. Stack & System Specifications

### Q: What is the embedding dimension and model used in Atrium?
Atrium generates **384-dimensional vector embeddings** for notes, chunk excerpts, and semantic centroids. By default, it uses ONNX embeddings via `fastembed` for semantic retrieval.

### Q: Can I run Atrium without local GPU hardware?
Yes. The local embeddings are compiled via ONNX and run on CPU threads. Audio transcription and image/PDF OCR default to Groq and Gemini APIs respectively, requiring minimal local resources.

## 2. Ingestion & Input Processing

### Q: What file formats are supported for document ingestion?
- **PDF Documents**: Parsed and chunked via PyMuPDF.
- **Images**: OCR processes screenshots, diagrams, and photos.
- **Web Pages**: Scrapes article titles, bodies, and meta tags.
- **Audio / Voice Notes**: Groq Whisper transcribes voice notes.

### Q: What is the 3-Ignore Pause Guard?
If a user ignores (does not answer) the custom context question prompt generated by the bot multiple times sequentially, the background worker activates a pause guard, suppressing prompts for the next few saves to prevent user notification fatigue.

## 3. Caching & Search Engine

### Q: How is search logic executed?
Atrium combines vector cosine searches and trigram text matches via Reciprocal Rank Fusion (RRF). 
- Cosine searches query the `embedding` vector column using HNSW index parameters.
- Text searches query the `summary` column using GIN trigram indexes.
RRF fuses both ranks together, and the synthesized query result is sent to the LLM.

### Q: What is Redis used for?
Atrium uses Upstash Redis for:
1. **Queuing**: Storing ingestion tasks under the list key `atrium:tasks`.
2. **Rate Limiting**: Track requests using sliding-window rate limiters.
3. **Graph Caching**: Storing computed Louvain community node clusters (`graph:{user_id}`).

---

## 4. Workspaces & Partnerships

### Q: What is the Hearth feature?
Hearth is a partnership system. Two users can link their workspaces using an invite code. Once connected, they share daily activity metrics, active hubs, and custom journey durations, backed by the `journey_pairs` and `journey_invites` database tables.
